# RWA 白皮书：真实资产上链的逻辑、现实与机会

> 作者视角：支付行业 CTO，传统金融 + Web3 双轨背景
> 写作时间：2026 年 3 月
> 定位：个人对 RWA 赛道的系统性思考，非投资建议

---

## 前言：为什么要认真研究 RWA

RWA（Real World Assets，真实世界资产Tokenization）不是一个新概念。2017 年就有人在做，2018 年国内叫"链改"，2021 年叫"资产上链"。每隔几年换个名字，再次被炒热，然后再次沉寂。

但 2024-2025 年这一波不一样。

不一样在于：**入场的人变了。** BlackRock、Goldman Sachs、Franklin Templeton、Fidelity、BNY Mellon——这些机构不是来"探索"的，他们在发真实产品，管理真实资金，接受真实监管。2025 年 Q3，Tokenization RWA 总规模突破 $300 亿（不含稳定币），较 2022 年增长 10 倍。

作为一个在支付行业工作超过 10 年、同时深度参与 Web3 支付基础设施建设的人，我写这篇文章，是要回答几个对我来说真正重要的问题：

- RWA 到底解决什么真实问题？哪些说法是炒作？
- 这个行业现在到底在哪个阶段？谁在赚钱？
- 对于支付行业的人来说，RWA 意味着什么机会？
- 我应该押注什么，回避什么？

这不是一篇综述报告，是我的判断。

## 第一章：拆解 RWA 的真实价值

### 1.1 先破一个最大的谎言

> "资产上了链就有流动性。"

这句话是 RWA 赛道里被重复最多、也最经不起推敲的话。

建一个网站不代表有流量。上链不代表有人买。**流动性来自于买卖双方的撮合，不来自于区块链本身。**

一个被Tokenization的卢森堡房产，如果全世界只有 50 个人知道它存在，它依然是一个流动性极差的资产——只不过它的所有权记录现在存在链上而不是纸上。

真正理解 RWA 的价值，需要先抛开"流动性神话"，回到本质：**Tokenization到底把什么事情变容易了？**

### 1.2 RWA 真正解决的三类问题

**问题一：私募市场的运营效率极低**

传统私募资产的交易流程：
- 找到资产：靠人脉和中介，信息极度不透明
- 尽职调查：需要律师、会计师，动辄数周
- 资金转移：电汇、银行对账、T+几天结算
- 所有权转让：纸质合同签署，转让流程可能需要数周

Tokenization之后：
- KYC/AML 自动化，不再需要每次人工验证
- 结算从 T+几天压缩到 T+0（秒级完成）
- 历史记录不可篡改，审计成本大幅降低
- 可以程序化设置合规规则（特定国籍、特定资产量才能持有）

这不是"更好的流动性"，这是**运营效率的数量级提升**。对机构来说，这直接降低了运营成本，减少了人工错误，加快了资金周转。

**问题二：高门槛资产的碎片化分配**

卢森堡一栋公寓楼的投资门槛可能是 100 万欧元。一个私募基金的最低认购可能是 500 万美元。这不是因为这些资产本质上需要这么高的门槛——而是因为**处理一个投资者的行政成本是固定的，和他投多少钱无关**。

具体来说，处理一笔私募投资，无论金额大小，都要走完这套流程：
- 收集投资者护照、地址证明 → 人工 KYC 审核
- 签署订阅合同 → 律师参与
- 银行电汇到账确认 → 人工对账
- 每月发送 NAV 净值报告 → 人工生成
- 分红/赎回时再走一遍 → 重复所有步骤

这套流程的人工成本大约在 300-500 欧元/人，于是出现了一个荒谬的算术：
- 投资者投 **100 万欧** → 行政成本占比 0.05%，可以接受
- 投资者投 **1000 欧** → 行政成本占比 30-50%，完全不划算

发行方的解法不是降低成本，而是**提高门槛**——直接把小投资者挡在门外。高门槛不是金融逻辑的必然，而是行政效率低下的副产品。

Tokenization 改变的是**成本结构**：用智能合约替代上述所有人工流程之后，处理 1 个投资者和处理 10,000 个投资者的边际成本趋近于零——**把固定的人工成本，变成了可以无限复制的代码成本**。门槛存在的经济理由消失了。

Tokeny 帮助卢森堡一个房地产项目实现了 **150 欧元起投**，让普通投资者（包括学生）也能参与一栋卢森堡市中心公寓楼的收益分配。这不是营销噱头，是技术真正改变了资产的可及性。

**问题三：持有资产而不出售的融资能力**

这是最被低估的 RWA 价值。

你持有 100 万美元的私募基金份额，但当前急需 20 万美元流动资金。传统金融的选择：要么出售份额（可能有损失，也可能有锁定期无法卖），要么去银行抵押贷款（流程复杂，时间长，门槛高）。

Tokenization资产可以做链上抵押借贷：
- 将持有的 Token 作为抵押
- 借出稳定币
- 不需要出售底层资产
- 全程自动化，无需银行审批

这个功能目前在消费端还没有大规模落地，但在机构间已经开始实践（DBS Bank 将 Tokenized MMF 用作抵押品进行 Repo 交易）。这是 RWA + DeFi 融合最有价值的方向。

### 1.3 RWA 解决不了的问题

公允地说，以下问题Tokenization并不能解决：

**流动性本身**：二级市场流动性仍然取决于买卖双方的匹配。目前绝大多数Tokenization资产的二级市场依然非常薄，上链没有改变这一点。

**底层资产的质量**：一个烂项目Tokenization之后还是烂项目。上链不能为资产的基本面背书。

**法律执行力的跨境问题**：Token 代表的权益在链上清晰记录，但如果发生纠纷，法院认的是合同，不是链上记录（至少目前大多数司法管辖区如此）。链上的所有权和链下的法律权益之间的对应，依然需要严谨的法律架构。

## 第二章：市场现状的真实图景（2025 Q3）

### 2.1 规模数据

| 资产类别 | Tokenization规模（2025 Q3） | 说明 |
|----------|----------------------|------|
| 私募信贷 | ~$170 亿 | 最大单一类别，占比约 57% |
| 美国国债 | ~$73 亿 | 增长最快，BlackRock BUIDL 占据 45% 市场份额 |
| 大宗商品（黄金/碳信用） | ~$20 亿 | 稳定增长 |
| 机构另类基金 | ~$20 亿 | 房地产、PE 等 |
| 合计（不含稳定币） | ~$300 亿 | 2022 年为 $29 亿，3 年 10 倍 |

**重要背景**：稳定币本身（USDT、USDC 等）如果算作Tokenization美元，体量超过 $2000 亿，已经是最成功的 RWA 形态。很多人讨论 RWA 时把稳定币单独剔除，但从本质上说稳定币就是Tokenization的法币——这是整个 RWA 叙事最强的 proof of concept。

### 2.2 增长驱动力分析

增长不是线性的，而是**政策驱动 + 机构示范效应**双轮驱动：

**政策节点**

- **SAB 121 废除（2025 年）**：此前美国会计准则要求持有数字资产的机构将其列为 Liability，导致银行无法为客户托管加密资产。废除后，机构托管数字资产的最大合规障碍消失。
- **Genius Act（2025 年）**：美国稳定币监管框架出台，机构使用稳定币进行结算有了明确的法律依据。
- **欧盟 MiCA 落地**：稳定币发行人的监管框架清晰化。
- **香港稳定币牌照推进**：亚洲最重要的金融中心开始接受受监管的稳定币。

**机构示范效应**

每一个大机构的实际行动，都在降低下一个机构入场的心理门槛：
- BlackRock BUIDL 启动（2024 年 3 月）→ 2025 年超过 $10 亿 AUM
- Goldman Sachs 发行 Tokenized MMF
- DBS Bank 接受 Tokenized MMF 作为 Repo 抵押品
- Binance 使用 Tokenized Treasury 做场外结算
- Nasdaq 申请上市Tokenization股票

这个模式就是：一个够大的玩家做了，其他人不得不跟进，否则就是落后。

### 2.3 谁在赚钱：价值链拆解

整个 RWA 价值链上，不同环节的赚钱逻辑差异很大：

```
发行层 → 技术层 → 合规层 → 分发层 → 交易层 → 管理层
```

**发行层（Issuer）**：发行人（如 BlackRock、Franklin Templeton）通过管理费赚钱，按 AUM 百分比收取，规模越大越赚。这是最稳定、最可持续的收入来源。

**技术层（Platform）**：Tokeny（SaaS License）、Securitize（一条龙服务）等。收入相对固定，订阅制为主，不直接与 AUM 挂钩，天花板明显。

**合规层（Legal/Custodian）**：律师、托管机构（如花旗、State Street）。一次性或年费收入，合规要求越严格越有议价空间。

**分发层（Distributor）**：券商、财富管理平台、DeFi 协议。赚分发佣金，类似传统金融的销售渠道费。

**交易层（Marketplace/Exchange）**：目前最薄弱的环节，但长期来看利润最大。类比 Crypto 交易所：交易量越大，手续费越多，不依赖单一资产涨跌。21X（欧盟监管）、tZERO 等在布局但还没有跑起来。

**管理层（Fund Administrator）**：Apex Group 这类。按 AUM 收管理服务费，Apex 收购 Tokeny 的逻辑就是：用技术平台做敲门砖，真正的收入来自后台管理服务。

**结论**：长期看，**AUM 管理费（发行层）和交易手续费（交易层）**是这个行业最大的两块蛋糕。技术层是基础设施，赚的是配套费，天花板受限。

## 第三章：技术架构的真实挑战

### 3.1 合规写进 Token：正确的方向

早期 RWA Tokenization有两种粗糙的做法：

**做法一：私有链**

只允许白名单节点访问，合规控制在链的准入层面。问题：无法与 DeFi 互通，无法接入公开交易所，本质上只是把数据库迁移到了区块链上。

**做法二：公链 + 钱包白名单**

资产在公链上，但通过平台维护白名单控制谁能持有。问题：合规逻辑依赖平台，平台宕机或被攻击则合规机制失效，本质上还是中心化控制。

**做法三（正确方向）：合规写进智能合约**

以 ERC-3643（T-REX）为代表，将 KYC/AML 规则直接嵌入 Token 本身：

```
传统模式：
投资者 → 平台白名单验证 → Token 转移

ERC-3643 模式：
投资者 KYC → 生成链上身份凭证（Claim）→ 绑定钱包
Token 转移时 → 智能合约自动调用 Identity Oracle → 验证双方身份
                                                    ↓
                                           合规 → 转移成功
                                           不合规 → 拒绝 + UI 提示
```

这个架构的核心优势：
- **去平台依赖**：Token 的合规状态不依赖任何中心化平台存活
- **可恢复性**：投资者丢失私钥，可通过发行方恢复 Token（私募资产不应该像比特币一样永久丢失）
- **可冻结/回收**：监管要求时，发行方可以冻结或没收特定钱包的 Token
- **可组合**：可以接入任何支持 ERC-3643 的 DeFi 协议、交易所

ERC-3643 已于 2022 年成为以太坊正式标准，140+ 机构参与协会，SEC 主席 Paul Atkins 公开提及，是目前最有机构认可度的合规 Token 标准。

### 3.2 尚未解决的技术问题

**Oracle 问题（最关键）**

链上 Token 的价值依赖于链下资产的定价。但链下资产的 NAV（净值）怎么上链？

- 谁来提供这个定价数据？
- 如果数据提供者出错或被操控，Token 价值直接被扭曲
- 目前的解决方案（Chainlink 等预言机）依然存在中心化风险

这是 RWA 体系中最深层的技术矛盾：**链上逻辑的可信度，依赖于链下数据输入的可信度。**

**跨链互操作性**

目前大多数 RWA Token 发行在特定链上（以太坊居多），但不同链之间的 Token 无法直接互通：
- 以太坊上的 BUIDL Token 不能直接在 Solana 上使用
- 每次跨链都需要 Bridge，Bridge 有安全风险和摩擦成本
- 机构投资者对跨链桥的安全性顾虑很大

这个问题没有被公开充分讨论，但实际上是阻碍 RWA 大规模流通的重要障碍。

**链上身份与链下法律的双重体系**

Token 代表的权益在链上是清晰的，但发生纠纷时法院认的是合同，不是链上记录。目前的解决方式是：链上 Token + 链下 SPV 公司 + 法律协议三层结构。

这个结构有效，但增加了复杂性，也意味着：**法律人才和技术人才的协同能力，是做 RWA 的核心门槛之一。**

## 第四章：主要玩家的真实定位

### 4.1 资产发行商（Issuer）

这一层做的是把传统金融产品搬上链，直接面向投资者。

**BlackRock BUIDL**
- 产品：Tokenized USD 货币市场基金（短期美债 + Repo）
- 发行在以太坊上，合作方 Securitize 负责技术
- 2024 年 3 月上线，2025 年 AUM 超过 $10 亿，占 Tokenized Treasury 市场 45%
- 意义：最大资管机构入场，行业最强背书

**Franklin Templeton（Benji）**
- 最早上线的机构级 Tokenized MMF（2021 年）
- 在 Stellar 和 Polygon 上发行，用自有平台 Benji
- AUM 约 $4 亿（2025 年）

**Ondo Finance**
- 纯 Web3 原生的 RWA 发行商
- OUSG（Tokenized US Treasury）、USDY（yield-bearing stablecoin）
- 面向 DeFi 用户，不要求机构 KYC 门槛
- 定位是把高质量链下收益带给 DeFi 用户

**关键判断**：Issuer 这层，传统机构（BlackRock、Franklin）靠品牌和 AUM 规模取胜，纯 Web3 玩家（Ondo）靠 DeFi 可组合性取胜，两者服务不同客群，不是直接竞争。

### 4.2 技术平台（Tokenization Platform）

这一层提供Tokenization的技术基础设施，不直接参与资产发行。

**Tokeny（欧洲，被 Apex 收购）**
- 最老牌的 RWA 技术平台，2017 年创立
- 创造了 ERC-3643 开源标准
- 核心优势：不锁定客户，开源标准，欧洲机构认可度高
- 被 Apex 收购后，技术平台 + Fund Administration 一体化

**Securitize（美国）**
- 既是技术平台，也是发行商（一条龙）
- BlackRock BUIDL 的技术合作方
- 更垂直整合，在美国合规框架下有先发优势

**两者本质区别**：Tokeny 是"工具提供商"，Securitize 是"服务提供商"。前者不锁定客户，后者提供全流程管理。

### 4.3 私募信贷平台（Private Credit）

这是目前 RWA 最大的资产类别（$170 亿）。

**Maple Finance**
- 链上机构借贷协议
- 借款方是经过 KYC 的机构（做市商、加密基金等）
- 贷款人是寻求高收益的机构或高净值投资者
- 问题：2022 年经历违约危机（Orthogonal Trading 违约），2023 年重建，转向更保守的借款方审核

**Centrifuge**
- 把真实世界的应收账款、贷款组合Tokenization
- 和 MakerDAO 深度合作，是最早的 RWA-DeFi 融合玩家
- 技术上更去中心化，但资产质量参差不齐是历史问题

**关键判断**：私募信贷 RWA 的核心风险不是技术，是**信用风险和尽调能力**。链上透明度提高了可见性，但并不能防止借款方违约。做好私募信贷 RWA，需要的是传统信贷机构的风控能力，而不只是区块链技术。

### 4.4 标准化资产平台（Tokenized Treasuries）

**Ondo Finance**（已在 4.1 介绍）

**OpenEden**
- 新加坡公司，面向机构投资者
- TBILL Token：基于美国国债的 yield-bearing stablecoin
- MAS 持牌机构，合规路径清晰
- 主要市场：亚洲机构投资者和 DeFi 用户

**关键判断**：这个赛道的护城河是**品牌信任 + 合规资质 + 流动性规模**。拼的不是技术创新，而是机构公信力。BlackRock 入场后，中小玩家的市场空间会被压缩。

## 第五章：监管格局与各地机会

### 5.1 美国：后发激进，政策主导全球

美国在 RWA 监管上历来保守，但 2024-2025 年发生了根本性转变：

**关键政策变化**
- **SAB 121 废除**：机构持有数字资产不再算 Liability，银行可以托管
- **Genius Act 通过**：稳定币获得联邦级监管框架，机构使用有法可依
- **SEC 态度转变**：Paul Atkins 接任 SEC 主席后，公开支持 RWA Tokenization
- **DTCC No-Action Letter**：链上结算获得监管放行，T+0 结算合法化

**影响**：每一次美国的政策松绑，都会倒逼欧洲和亚洲市场加快节奏。全球 RWA 监管的节拍器在华盛顿。

### 5.2 欧洲：先发但受限，改革中

欧洲在立法上是全球最早的（MiCA、DLT Pilot Regime），但具体的 DLT 试点框架对创新有很大限制：

- 只允许三类资产：股票（市值 ≤ €5 亿）、债券、UCITS 基金
- 二级市场交易量上限：€60-90 亿（远远不够）
- 牌照有效期仅 6 年
- 申请成本极高，挡住中小创新公司

结果：欧洲在立法上领先，但实际落地规模被美国赶超。已有多家欧洲机构联名致信欧盟要求放开限制。

**卢森堡的特殊地位**：在欧洲金融监管中以开放著称，率先通过区块链相关法律，是欧洲 RWA 创新的事实中心。

### 5.3 新加坡：最务实的亚洲中心

新加坡 MAS 的监管策略是：**先跑通案例，再形成标准**。

- **Project Guardian**：MAS 联合多家机构（DBS、JPMorgan、Citi）探索Tokenization资产的跨境交易，已推荐 ERC-3643 标准
- **持牌制度**：新加坡对数字资产的持牌体系相对清晰，OpenEden 等公司已持牌运营
- **机构接受度**：DBS Bank、星展等传统金融机构已深度参与 RWA 试点

**判断**：新加坡是亚洲 RWA 机构化程度最高的市场，政策确定性最强，是亚洲 RWA 机构的首选注册地。

### 5.4 香港：追赶中，重点在稳定币

香港 2024-2025 年大力推进 Web3 友好政策，但 RWA 具体的落地框架还在完善中：

- 稳定币牌照制度：港元稳定币的发行框架在推进，有望成为首个发行监管稳定币的主要经济体之一
- SFC 对Tokenization证券的态度：已接受 ERC-3643 等合规标准的深度讨论
- 挑战：合规要求细节多，执行层面比新加坡复杂

**判断**：香港更适合做**稳定币支付 + RWA 结算**的结合场景，而不是纯 RWA 发行。

### 5.5 中国大陆：监管套利空间极小

国内 42 号文明确：

> 境内机构和个人不得从事虚拟货币相关业务，包括Tokenization资产的发行和交易。

**但留了一个口子**：
> "经业务主管部门依法批准的，依托特定基金基础设施开展的相关活动除外。"

现实判断：这个口子极窄，需要主管部门逐案批准，且必须依托"特定基金基础设施"（即国家认可的金融基础设施，如上交所、深交所旗下平台）。对于市场化团队来说，短期内几乎没有可操作空间。

**策略**：有大陆业务的团队，RWA 方向应该放在新加坡或香港法律实体下，不要试图在大陆走 RWA 路线。

## 第六章：稳定币与 RWA 的协同效应

### 6.1 为什么稳定币是 RWA 最大的催化剂

这是目前市场上最被低估的洞察。

**现状的荒谬**：很多机构已经把资产Tokenization了，但结算还在用银行电汇。你Tokenization了基金份额，但结算的时候还是 T+2，等银行放款——整个Tokenization的意义大打折扣。

原因是：机构之前不能在 Balance Sheet 上持有稳定币（因为 SAB 121 要求计为 Liability），所以只能用传统银行电汇来结算Tokenization资产的交易。

**SAB 121 废除 + Genius Act 通过之后**，机构可以合法持有和使用稳定币，整个链条才真正打通：

```
链上资产（Tokenized Fund / Bond / Real Estate）
           +
链上现金（Regulated Stablecoin / CBDC）
           =
自动化即时结算（T+0，24 小时，无需银行）
+ 链上抵押借贷（持有资产不卖就能借钱）
+ DeFi 可组合（接入 Uniswap、AAVE 等协议）
```

这不是未来的愿景，2025 年已经在发生：
- Binance 使用 Tokenized Treasury 做场外结算
- DBS Bank 使用 Tokenized MMF 做 Repo 抵押品
- 多个 DeFi 协议接受 BUIDL 作为抵押品

### 6.2 Tokenized Money Market Fund：当前最成功的 RWA 形态

Tokenized MMF（如 BlackRock BUIDL、Franklin BENJI）之所以成为 RWA 落地最成功的产品，原因是它同时满足了三种需求：

**对机构来说**：
- 链上"余额宝"：持有 Token 就自动赚取美债利率（当前 ~5%）
- 合格抵押品：可以用作 DeFi 协议的抵押，比持有 USDC 多一层收益
- 结算工具：机构间资金结算时，不用持有零利息的 USDC，改用 BUIDL，资金效率更高

**对 DeFi 来说**：
- 引入了真实世界的高质量资产作为抵押品，降低了 DeFi 系统性风险
- 使用传统金融的信用背书（BlackRock 品牌），吸引机构用户进入 DeFi

**对监管来说**：
- 底层是最传统、最合规的资产（美国国债）
- 发行方是受严格监管的机构（BlackRock）
- 合规框架清晰，监管可以接受

这就是为什么 Tokenized MMF 在过去 18 个月增长了 **10 倍以上**，而其他资产类别增长更慢。

### 6.3 支付行业的 RWA 机会

从支付行业视角，RWA 有几个非常直接的机会点：

**备付金的收益化**

支付公司（包括发卡机构、收单机构）持有大量备付金，传统上只能放在银行低利率账户。Tokenized MMF 提供了一个合规的替代方案：
- 将备付金配置为 BUIDL 或同类产品
- 自动赚取约 5% 的美债收益
- 同时保持 T+0 的流动性（Tokenized MMF 支持即时赎回）
- 在新加坡/香港监管框架下完全可行

**稳定币收单的结算层升级**

目前稳定币收单的结算逻辑：商户收到 USDC → 兑换法币 → 结算给商户。

引入 Tokenized MMF 之后的逻辑：商户收到 USDC → 自动配置为 BUIDL → 每天自动产生利息 → 商户按需赎回法币。这让稳定币收单从"零成本通道"变成了"带收益的资金管理服务"，商业模式升级。

**链上 KYC 与身份基础设施**

ERC-3643 的 Identity Oracle 机制，本质上是一套链上 KYC 体系。对于已经建立了完整 KYB/KYC 体系的支付公司来说，这是可以复用的资产：
- 一次 KYC → 生成链上身份凭证 → 用户在任何支持该标准的平台上无需重复 KYC
- 这是支付合规基础设施的未来形态

## 第七章：我的判断与行动框架

### 7.1 哪些叙事成立，哪些是炒作

**成立的：**
- 私募市场运营效率提升（结算、KYC 自动化）✅
- Tokenized MMF 作为链上高质量抵押品和结算资产 ✅
- 稳定币 + RWA 结合打通整个链上金融体系 ✅
- 机构主导的合规 RWA 会持续增长 ✅

**部分成立，需要谨慎：**
- "RWA 上链就有流动性"——⚠️ 错误，流动性仍需要买卖双方撮合
- "私募市场比公开市场大数倍"——⚠️ 数字不准确，但私募市场痛点更深是对的
- "开源标准能统一行业"——⚠️ 多标准并行是现实，ERC-3643 在欧洲领先，Securitize DS Protocol 在美国也有地位

**炒作，短期内不成立：**
- "所有股票债券都会被Tokenization"——太早，法律体系没准备好
- "散户能参与 RWA 赚高收益"——监管要求、门槛和用户教育都是障碍
- "RWA 会颠覆传统金融"——更可能是渐进融合，不是颠覆

### 7.2 行动优先级

**现在就应该做的：**

- 研究 Tokenized MMF 的法律框架（新加坡/香港），评估备付金配置可行性
- 跟踪 Genius Act 具体条款对稳定币收单业务的影响
- 了解 ERC-3643 架构，思考链上 KYC 与现有 KYC 体系的融合路径

**6-12 个月内跟进的：**

- MAS Project Guardian 下一阶段成果
- 香港稳定币牌照落地后的具体监管要求
- BUIDL 是否开放更低门槛的机构接入

**不需要现在押注的：**

- 国内 RWA：合规路径太窄，优先级低
- Tokenized 公开股票：标准战争还没结束，观望
- 私募信贷 RWA：需要专业信用风控能力，非我方核心优势

### 7.3 对 RWA 整体的判断

RWA 不是一次技术革命，而是一次**金融基础设施的渐进升级**。

它的本质是：用区块链的可编程性和透明度，来改造传统金融市场效率极低的基础设施层（结算、KYC、所有权记录）。它不会颠覆传统金融，它会被传统金融机构吸收。

这个判断意味着：
- 纯 Crypto Native 的 RWA 项目，最终会被机构版本替代或收购（就像 Tokeny 被 Apex 收购）
- 有监管资质 + 技术能力的混合团队，是最有竞争力的
- 做基础设施（合规标准、结算层）比做具体产品更有长期价值
- 支付行业是这个升级过程中天然的受益者：支付是资金流动的入口，链上金融的兴起会让支付基础设施更加重要

最后一个判断：**稳定币会先于 RWA 大规模普及**，而稳定币的大规模普及会反过来大幅加速 RWA 的落地。这两条线不是平行的，而是相互催化的。谁在稳定币支付里有了根基，谁在 RWA 里就有了先发优势。
