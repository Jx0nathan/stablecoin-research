## 二、核心命题：RWA 的价值在哪里？

### 2.1 最大的误区：上链 ≠ 流动性

> "上了链就有流动性"——这是 RWA 赛道被重复最多、也最经不起推敲的一句话。

类比建网站：建了网站不代表有流量，上了链不代表资产自动热销。**区块链解决的不是流动性问题，而是私募市场的运营痛点。**

### 2.2 RWA 真正解决的三个问题

**① 降低私募市场门槛**

传统私募资产门槛极高（100 万～500 万欧元起），流程全靠人工，中间商系统各自独立。通证化后：

- 最低投资额可降至 150 欧元（卢森堡房地产案例）
- KYC/AML 全程自动化
- 覆盖更多小型富裕投资者（Family Office、高净值散户）

**② 链上抵押借贷**

持有传统私募资产但需要流动资金，只能出售资产。通证化后可以：

- 将持有的 Token 作为抵押，借出稳定币
- 不需要出售底层资产
- 这是传统金融无法实现的功能

**③ 结算效率**

传统私募基金份额转让：T+几周（甚至更长）

Tokeny 通证化后：**几秒钟**完成

Skybridge Capital（美国对冲基金）案例：通证化基金份额，结算时间从数天压缩至秒级，他们称之为"革命性"。

## 三、技术架构：合规写进 Token

### 3.1 行业演进三阶段

**阶段一：私有链（Private Blockchain）**

- 只允许特定节点访问，像局域网
- 优点：发行方完全控制
- 缺点：无法接入 DeFi，无法连接第三方交易所，互通性极差

**阶段二：公链 + 钱包白名单**

- 资产上公链，通过白名单控制谁能持有
- 优点：可接入 Uniswap 等 DeFi 协议
- 缺点：合规数据仍在链下平台，平台宕机或被攻击则合规逻辑失效，本质上依赖中心化平台

**阶段三：合规直接写入 Token（ERC-3643）**

- KYC/AML 规则编码进智能合约本身
- 通过链上 **Identity Oracle**（身份代理合约）验证投资者资格
- Token 转让时自动验证双方合规状态，无需平台参与

### 3.2 ERC-3643（T-REX）核心机制

```
投资者 KYC → 生成链上身份凭证（Claim）→ 绑定钱包地址
                                           ↓
Token 转让时 → 智能合约自动调用 Identity Oracle → 验证双方身份凭证
                                                   ↓
                                           合规 → 转让成功
                                           不合规 → 拒绝 + UI 提示完成 KYC
```

**关键特性：**
- 一个身份合约可绑定多个钱包（换钱包不影响资格）
- 私钥丢失可通过发行方恢复 Token（不像比特币永久丢失）
- 发行方可冻结、回收 Token（满足监管要求）
- 可设置任意合规规则：投资者国籍、人数上限、转让验证方式等
- 完全兼容 ERC-20，在其上加一层 Validation System

### 3.3 与 Securitize 的本质区别

| 维度 | Tokeny | Securitize |
|------|--------|------------|
| 角色定位 | 技术提供商（工具层） | 发行人（一条龙服务） |
| 商业模式 | SaaS License | 发行 + Marketplace 抽成 |
| 智能合约 | ERC-3643 开源标准 | DS Protocol（自有协议） |
| 互操作性 | 行业标准，任何人可用 | 相对封闭 |
| 发行决策 | 客户自己决定发什么资产 | Securitize 主导 |

Tokeny 的核心优势：**不锁定客户**。用 ERC-3643 发行的 Token，未来可以接入任何合规交易所、DeFi 协议，不被单一平台绑架。

## 四、ERC-3643 生态

### 4.1 标准化历程

- Tokeny 内部标准（T-REX）→ 2021 年提交 ERC 提案 → 两年审核修改 → **2022 年正式成为以太坊 ERC 标准**
- 同步成立 ERC-3643 非营利协会，推动开源标准行业化

**开源的战略逻辑：**
> "如果不开源，蛋糕只有 Tokeny 吃，但只能吃到很小的一块。如果整个行业用同一个标准，蛋糕变大了，我们作为标准提供者能分到大块。"

这是典型的平台战略——不靠垄断技术赚钱，靠成为行业基础设施赚钱。类比 Apple 生态。

### 4.2 生态成员（140+）

核心成员包括：
- 基础设施：Chainlink、Polygon、Avalanche
- 金融机构：Invesco、ABN Emerald、Apex Group
- 专业服务：Deloitte（德勤）、华兴资本（Pinpoint）
- 交易所：21X（欧盟监管数字资产交易所）

**监管层认可：**
- SEC 主席 Paul Atkins 已多次公开演讲中提及 ERC-3643
- 卢森堡相继出台约 4 部区块链法律，为欧洲首批明确通证化合规框架的国家
- 新加坡 MAS Project Guardian 推荐了 T-REX 标准
- 香港 SFC 已有过深入了解和讨论

## 五、典型案例

### 案例一：Skybridge Capital（美国对冲基金）
- **痛点**：基金份额 A 转 B，传统结算需要数天
- **解决方案**：通证化基金份额，结算压缩至秒级
- **后续规划**：接入 DeFi 功能，实现更多链上操作

### 案例二：Blockhome（卢森堡房地产众筹）
- **背景**：2019 年接触，卢森堡房地产投资门槛极高
- **创新**：通过 SPV 结构，最低 150 欧元即可参与
- **监管合作**：与 CSSF（卢森堡 SEC）深度沟通，监管提出三点要求：
  1. 合规性验证
  2. 链上资产可控（只卖给合法投资者）
  3. 资产恢复机制（私钥丢失可找回）
- **结果**：募集约 800 万欧元，购买卢森堡市中心地块并建成出租住宅楼
- **二级市场**：平台内 Peer-to-Peer 转让，买卖双方均需通过合规验证
- **政策影响**：推动卢森堡出台首批区块链通证化合规法律

### 案例三：新加坡货币市场基金
- 通证化 Money Market Fund 份额
- 实现即时买卖，创始人在行业内资深
- 低调运营，未公开宣传

### 案例四：香港应收账款（Receivable）通证化
- 与当地银行合作，探索应收账款（Invoice）的通证化
- 目标：加速应收账款的资本流动性
- 时间：约 2-3 年前，未公开详情

## 六、市场格局与竞争

### 6.1 私募市场 vs 公开市场

Tokeny 坚持聚焦私募市场，理由：
- 公开股票市场流动性已经很好，通证化增量价值有限
- 私募/另类资产市场规模是公开市场的数倍（估计占总资产 80%+）
- 私募市场痛点真实：手动流程、高门槛、合规复杂

**对公链 Tokenized 股票的判断：**
> 对于原本就能直接投资股票的用户，通证化并没有带来什么更好的功能，反而多了一个中间商和费用层。真正的价值在于抵押借贷功能，但目前还没有被真正实现。

### 6.2 谁吃最大的蛋糕

按 Shurong 的判断，价值链分配如下：

```
最赚钱：
① Marketplace（交易场所） → 类比 Crypto 交易所，靠 Transaction Fee，市场越活跃越赚
② AUM 管理费 → 和资产规模直接挂钩，可持续

中等：
③ Transfer Agent + Fund Administrator → Apex 的核心收入来源

一次性：
④ 律师（法律文件） → 每次发行一次性费用
⑤ Tokenization Platform（SaaS License） → 订阅费用，相对固定
```

**Apex 收购 Tokeny 的逻辑：** Tokeny 是"敲门砖"，帮 Apex 拿到技术入口，真正的收入来自 Fund Administration 服务费（按 AUM 百分比收取）。

## 七、监管环境分析

### 7.1 欧洲：先发但受限

**DLT Pilot Regime（欧盟 DLT 试点制度）：**
- 只允许三类资产：股票（市值 ≤5 亿欧元）、债券、UCITS 基金
- 二级市场交易量上限：60-90 亿欧元
- 牌照有效期：6 年
- 高昂的申请成本，将初创公司挡在门外

多家欧洲公司已联名致信欧盟要求松绑，因为美国已给出 T+0 即时结算许可，资产有流向美国市场的风险。

### 7.2 美国：后发但激进

关键政策节点：
- **SAB 121 废除**：此前要求持有数字资产计为 Liability，导致银行无法托管加密资产。废除后彻底打开机构托管口子。
- **Genius Act**：稳定币监管框架出台，打开机构使用稳定币的合法通道
- **SEC 非干预声明（No-Action Letter）**：给 DTCC 的链上结算放行，欧洲公司感到巨大压力

### 7.3 亚洲：加速追赶

| 地区 | 状态 |
|------|------|
| 新加坡 | Project Guardian 推荐 ERC-3643，MAS 开放态度，机构接受度高 |
| 香港 | 稳定币牌照推进中，SFC 对 ERC-3643 有深入了解 |
| 韩国 | 加密接受度极高（两人中一人有钱包），最热衷 IP 通证化（K-drama/K-pop） |
| 马来西亚 | 房地产和私募股权项目最多 |
| 中国大陆 | 42 号文：境内原则禁止，但"经主管部门批准、依托特定基金基础设施"除外 |

**对 42 号文的解读：**
> 看起来是"一杆子打死"，但仔细看留了一个口子："经业务主管部门依法批准的，依托特定基金基础设施开展的相关活动除外。" 这给了在监管框架内操作的空间。

## 八、稳定币与 RWA 的协同效应

这是 Shurong 最核心的前瞻判断：

**现状问题：**
- 很多机构使用 Bank Transfer 买卖通证化资产
- 没有真正体验到通证化的价值（结算依然慢）
- 原因：SAB 121 导致机构不愿在 Balance Sheet 持有稳定币

**打通后的未来：**

```
链上资产（Tokenized Fund / Bond / 房产）
         +
链上现金（Regulated Stablecoin / CBDC）
         =
自动化即时结算（T+0）+ 链上抵押借贷 + 24 小时交易
```

> "稳定币是推动整个市场最大的 Booster。"

**Tokenized Money Market Fund 的特殊地位：**
- 既是"链上余额宝"（24 小时赚利息）
- 又是高质量抵押品（用于 DeFi 协议、Repo 交易）
- BlackRock BUIDL 案例：2.4 Billion AUM，验证了 Product-Market Fit

## 九、对 DCS / 支付赛道的启示

### 9.1 直接机会：稳定币收单的 RWA 背书

DCS 当前做 U 卡和稳定币收单，RWA 的机会在于：

- **稳定币备付金管理**：将备付金配置为 Tokenized Money Market Fund，既保持流动性，又赚取收益
- **商户结算资产**：商户收到的稳定币可以直接在链上配置为 RWA，不用手动操作

### 9.2 合规技术借鉴：ERC-3643 的 KYC 模式

ERC-3643 的 Identity Oracle 机制与支付的 KYC 需求高度重合：
- 一次 KYC → 生成链上凭证 → 多个场景复用
- 这是支付合规的未来形态，值得跟踪

### 9.3 竞争格局认知

Tokeny（技术层）vs Securitize（一条龙）vs Ondo（标准化资产发行）三条路径分别对应不同的市场位置：
- Tokeny：基础设施，服务机构客户，不锁定客户
- Securitize：垂直整合，适合美国市场的合规发行
- Ondo：聚焦高流动性公开资产（美债），走量不走深

支付公司（如 DCS）更应该关注的是 **Tokeny 模式**：做链接稳定币和 RWA 的基础设施层，而不是自己去做 RWA 发行。
