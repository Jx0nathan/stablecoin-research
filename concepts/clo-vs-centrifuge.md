# CLO vs Centrifuge：传统结构化信贷与链上实现对比

## 一、什么是 CLO

CLO（Collateralized Loan Obligation，担保贷款凭证）是传统金融中的一种结构化融资产品。

**核心逻辑：把一批贷款打包，分层卖给不同风险偏好的投资者。**

### 参与方

- **资产发起方（Originator）**：银行或贷款机构，持有一批企业贷款
- **SPV（特殊目的载体）**：独立法律实体，从发起方购买贷款资产，实现资产隔离
- **投资者**：按风险偏好认购不同层级的证券

### 资产结构（Tranching / 分层）

```
贷款池（100）
    ↓
┌─────────────────────────────┐
│ Senior Tranche（优先级）70% │  → 最先被偿还，收益低，风险低（AAA 评级）
├─────────────────────────────┤
│ Mezzanine Tranche（中间层）20% │  → 次级偿还，收益中等
├─────────────────────────────┤
│ Equity / Junior Tranche 10%│  → 最后被偿还，承担首损，收益最高
└─────────────────────────────┘
```

### 资金流向

```
投资者出资
    ↓
SPV 用资金从发起方买入贷款资产
    ↓
贷款人还款（利息 + 本金）流入 SPV
    ↓
按层级优先顺序分配给投资者
（Senior 先拿，Equity 最后拿）
```

### 关键机制

- **超额抵押**：贷款池价值 > 发行证券总额，提供缓冲
- **瀑布式分配（Waterfall）**：现金流严格按优先级顺序分配
- **信用增强**：通过分层结构，Senior 层可获 AAA 评级，即使部分贷款违约也不受损

## 二、Centrifuge 如何把 CLO 搬上链

Centrifuge 复制了 CLO 的核心逻辑，用智能合约替代 SPV，用 NFT 替代贷款凭证，用 ERC-20 Token 替代证券。

### 角色映射

| 传统 CLO        | Centrifuge          |
|----------------|---------------------|
| 资产发起方      | Asset Originator（池子创建者）|
| SPV 法律实体    | 智能合约资金池（Pool）|
| 单笔贷款凭证    | NFT（每笔资产铸造一个）|
| Senior 证券     | DROP Token（固定收益，优先偿还）|
| Equity 证券     | TIN Token（浮动收益，承担首损）|
| 瀑布式分配      | 合约内置分配逻辑|
| 评级机构        | 无（投资者自行评估）|

### 链上资金流向

```
投资者存入 DAI/USDC
    ↓
资金池智能合约
    ↓
资产发起方借出资金（以 NFT 作为链上抵押凭证）
    ↓
发起方还款（链上转账）
    ↓
合约按瀑布逻辑分配：
    → DROP 持有人优先获得固定利息
    → TIN 持有人获得剩余收益（或承担亏损）
```

### 核心合约逻辑

```
Pool（资金池）
  ├── Shelf（贷款管理）：管理 NFT 作为抵押品
  ├── Pile（利息计算）：跟踪每笔贷款的利率和还款
  ├── Tranche（分层）：管理 DROP/TIN 份额铸造和赎回
  └── Assessor（估值）：计算池子 NAV，确定 DROP/TIN 价值
```

## 三、关键差异

### Centrifuge 比传统 CLO 好在哪里

- **结算效率**：链上自动结算，T+0，无需清算所中介
- **透明度**：所有还款、分配、NAV 变动链上可查
- **可组合性**：DROP Token 可接入 DeFi（如 MakerDAO），用于抵押借款
- **准入门槛低**：中小企业可绕过传统银行承销流程直接发起资金池

### Centrifuge 比传统 CLO 差在哪里

- **法律强制力弱**：NFT 的法律效力在多数司法管辖区未经测试
- **违约处置难**：链下资产违约时，链上合约无法强制执行债务追偿
- **评级缺失**：没有评级机构背书，机构投资者难以参与
- **流动性差**：DROP/TIN 的二级市场几乎不存在

## 四、适用场景对比

| 场景                    | 传统 CLO | Centrifuge |
|------------------------|---------|------------|
| 大型银行贷款池（$1B+）   | ✅      | ❌（规模限制）|
| 中小企业发票融资         | ❌（门槛高）| ✅        |
| 需要 AAA 评级的机构产品  | ✅      | ❌         |
| 需要 DeFi 可组合性       | ❌      | ✅         |
| 新兴市场中小贷款机构     | ❌      | ✅         |

## 五、与其他 RWA 产品的定位区别

```
Ondo / OpenEden / Franklin BENJI
    → 标准化资产（国债/货币市场）+ ERC-20
    → 面向零售/机构投资者
    → 不需要分层结构

Centrifuge / Maple Finance / Goldfinch
    → 非标私人信贷（发票/贷款/信用额度）+ NFT + 分层
    → 面向寻求高收益的 DeFi 资金
    → CLO 链上化

两者解决的是完全不同的问题，不存在替代关系。
```
